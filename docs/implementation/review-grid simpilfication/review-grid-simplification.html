Review Grid Architecture (Current)

Two tables, two actions

TableRole
productsEvery product per category (seeded from product.json)
field_candidatesAll candidates, sources, confidence. status='resolved' = published value.

Data flow

Pipeline extracts value
  → submitCandidate()
  → validateField() rejects bad data at the gate
  → if confidence meets threshold → publishCandidate() marks resolved
  → Done. No human accept step.

Override:
  User clicks candidate or types value in drawer
  → submitCandidate() with confidence 1.0
  → auto-publishes, grid updates optimistically

Grid cell states

StateMeaningColor
Has resolved candidatePublished value, show confidence + formatted valuegreen/yellow
No resolved candidateEmpty field, confidence shown if > 0gray
Product not runColumn dimmed, no datadimmed
