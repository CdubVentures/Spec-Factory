Drawer Panel — Target Design

Target state for the CellDrawer after full cleanup. Status updated 2026-04-11.

Core principle: No accept, no confirm, no flags, no keyboard shortcuts. The pipeline auto-publishes. The resolved candidate IS the published value. The only user actions are override (click candidate or type value) and AI review (future).

Interaction Model

  1. Click a cell → opens drawer + starts editing
  2. Click a candidate card → overrides the published value
  3. Type in Manual Override → same flow, manual entry
  4. AI Review → future, not connected yet

No accept step. No confirm step. No flags. No lanes. No keyboard shortcuts.

Sections

  1. Published Value — the resolved candidate. Confidence dot + value + source. OVR badge if manual override.
  2. Manual Override — text input + Apply.
  3. Candidates — clickable cards. The published source is marked. Click any other to override.
  4. AI Review — button placeholder (future).

Current props (field review grid)

  • title, subtitle, onClose
  • currentValue (value, confidence, color, source, sourceTimestamp, overridden, acceptedCandidateId)
  • badges (always [] for field review)
  • onAcceptCandidate(candidateId, candidate)
  • onManualOverride(value)
  • isPending
  • candidates, candidatesLoading
  • onRunAIReview, aiReviewPending

Dead props removed DONE

14 dead props removed. Remaining component-review props (isCurrentAccepted, pendingAIShared, onConfirmShared, onConfirmSharedCandidate, candidateUiContext, showCandidateDebugIds) are scheduled for retirement in Component Review Convergence.
