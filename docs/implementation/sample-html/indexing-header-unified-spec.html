Indexing Panel Header — Unified Spec

One reusable header component for Pipeline · CEF · PIF · RDF · SKU · Key. Kills drift. Adds per-panel color, expandable state, model chips, and a strict button matrix.

v1 · 2026-04-21 · spec + anatomy

1 Why this exists

The six indexing tabs (Pipeline + 4 finders + Key) currently render six different headers. The drift is structural, not cosmetic — Key uses a bespoke gradient toolbar, Pipeline hand-writes its own layout, CEF/PIF/RDF/SKU share a partial component but fill it inconsistently.

Current drift summary Three separate header implementations across six panels. Only the Key panel shows product id (not needed). Only CEF exposes the Prompt button today. Model badges appear in four panels but not in Pipeline or Key. The .finder-tab-icon-key color class is missing from FinderTabBar.css.

6 Expandable behavior

The header row itself does not change between collapsed and expanded — only the body below it appears/disappears. The chevron flips and the panel's specialized content (KPI strip, phase progress, run controls, filter row) slides in.

Key panel becomes compliant The dark blue gradient bar in KeyFinderToolbar becomes the indigo rail + icon of the unified header. The KPI strip and filter row become the expanded body. No more product-id display, no more bespoke layout, no more missing chevron.

7 Consistency rules

DO

  • Every panel renders the same IndexingPanelHeader component — no exceptions.
  • Panel identity is passed via panel prop; all styling reads from tokens.
  • Model strip data comes from the resolved llm-config, not hardcoded names.
  • Add new panels by adding one token trio + one button-matrix row — nothing else.
  • Register the key color trio + .finder-tab-icon-key CSS class before Key migrates.

DON'T

  • Don't hand-build a panel header. If a preset doesn't fit, propose a new preset — don't fork.
  • Don't show product id anywhere in the header. Product id belongs in the global product switcher.
  • Don't hardcode #4263eb or any hex in the header — use panel tokens.
  • Don't put run state in the title ("Pipeline · Running"); use the Running pill slot.
  • Don't render a Prompt button for Pipeline or Key. Don't render History for Pipeline.
