1 What changed in this passdelta vs. draft v1

Seven polish moves
  • Shared rail column (108px). Every band uses a 3-column grid: rail · content · trail. Axis labels (DIFFICULTY, AVAILABILITY, REQUIRED, STATUS) and the QUICK label all align on the same right-justified x, so segmented groups start at a single column down the page.
  • Softer chip on-states. Selected chips shift to a tinted fill (bg-token + border-token + color-token text) instead of saturated color floods. Readable, quieter, still unambiguous.
  • Unified icon set. Inline SVG symbols replace mixed emojis (🔍, ⤢, ⤡, ×). Icons share stroke weight, color, and 12–14px sizing.
  • Consistent control height (28px) for every chip, segment, and tool button. Eliminates the ragged vertical edge the earlier pass had.
  • Hairline dividers. Bands separated by border-top: 1px solid var(--border-light), no background shift except the axes band (subtle surface-soft sink).
  • Count style simplified. Inline separator (· replaced by a thin left-border divider) instead of nested pill-in-pill. Reads as tabular numerals without the nested-badge noise.
  • Zero-count chips dim to 50%. If an axis value has 0 matches, its chip loses weight but stays in place — keeps alignment stable as data flows.

5 Anatomy & alignment

Every band is a grid-template-columns: var(--rail-w) 1fr auto. The rail is a 108px right-justified label column. Content always begins at the same x; trailing tools always align to the right edge.

Rail column

  • Fixed width --rail-w: 108px, right-aligned text.
  • Band 1 (search) uses an empty rail — keeps the search input starting at the same x as every chip row beneath it.
  • Type style: 10px / upper / +.7px letterspacing / --text-dim.

Content column

  • flex-wrap:wrap with 8px gap — chips reflow gracefully at narrow widths.
  • First band holds search + meter side-by-side; remaining bands hold one preset row or one segmented group.
  • min-width:0 prevents long searchbox from pushing the trail off-screen.

Trail column

  • Only populated on band 1 (Expand / Collapse / Clear / info).
  • Other bands leave it empty so content and trail columns consistently flank the rail — stable geometry band-to-band.
  • Buttons share a 28px height and 6px gap — same rhythm as chips on their left.

Band backgrounds

  • Band 1 (search) and band 2 (presets) sit on --surface — primary filter surface.
  • Bands 3–6 (axes) shift to --surface-soft — a faint sink grouping the facet rows visually.
  • Dividers are 1px --border-light only — no chunky fills between rows.
