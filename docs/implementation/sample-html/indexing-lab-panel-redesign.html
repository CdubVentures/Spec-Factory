Indexing Lab — Panel Redesign Preview

Problem today: The Indexing Lab panel reads as a flat, long scroll of four near-identical finder accordions. You can't tell at a glance which tab you're on, which product you're working on, what's running, or what the primary next action is. Per-variant actions are encoded as tiny 3-letter chips (CLR, VIEW, HERO, LOOP, EVAL) with no legend.

This redesign: adds an orientation strip, a mission bar, a 2×2 finder dashboard that fits above the fold, an expand-in-place detail drawer per finder, labeled action buttons, and a chip legend. Same data, same finders, same actions — re-prioritised so the active context and next step are always obvious.

1App-level Orientation — tab strip + page header
Change: The active tab gets an underline + bold weight (today it's only a thin border). Below the tabs, a page header gives you a title, a one-line purpose, and a persistent run-status pill that is visible whether or not any finder section is expanded.

03 Evaluate Vision model ranks candidates; best are assigned to carousel slots. View eval first, then hero eval.
04 Publish Resolved images are written to the SQLite projection so they appear in Review Grid.

Summary — what this redesign changes

Before · current panel

  • No page identity. You land in a sea of accordions with no "Indexing Lab" heading.
  • Three-box product picker dominates the top even after you've selected a product.
  • Three stacked full-width buttons (Run, Stop, Clear / Replay) fight for attention every render.
  • Four identical finder accordions stacked vertically; you scroll to find the one you care about.
  • Cryptic chips (EGT, EVAL, CLR, HERO, LOOP, FB) with no legend.
  • Tiny 3-letter action buttons per variant with ambiguous intent.
  • Run status is invisible unless you look at the big red Stop button.

After · this redesign

  • Page header with title, purpose, and a persistent run-status pill.
  • Mission Bar collapses product context into a compact strip with one primary button.
  • 2×2 finder dashboard fits all four finders above the fold with a status light, primary KPI, and one primary action each.
  • Detail drawer with tabs (Overview / Variants / Runs / How It Works) opens inline when you click a card.
  • Labeled per-variant actions (Views / Hero / Loop / Evaluate / Clear) replace 3-letter chip buttons.
  • Chip legend at the bottom + tooltip on hover — no more guessing.
  • Run status pill shows idle / running / stage at all times.
