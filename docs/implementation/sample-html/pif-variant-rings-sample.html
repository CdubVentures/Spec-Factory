Three fat segmented rings per variant. No bonus — segment dividers carry the count.

Legend
Outer — Priority Views (Single Run) Middle — Hero slots Inner — Loop Run extras (viewBudget minus priority) Amber = partial / in-progress
Each ring is segmented: divided into N arcs where N = that ring's target. With stroke-width: 5 and ~3.5px gaps, the count is readable at a glance — no external label needed. Inner ring also gets dividers when its target > 1.

Per-ring contract

  • Outer r=21 — Priority Views (Single Run) = viewConfig entries with priority:true
  • Middle r=14 — Hero Slots = heroCount when heroEnabled
  • Inner r=7 — Loop Run extras = viewBudget views NOT in priority
  • Ring goes amber while partial, green/orange/blue when each ring's full target is met
  • Color chip: 18×18 rounded square above; gradient supports multi-atom colors
  • Fraction label X/Y under = sum across all 3 rings (hover tooltip breaks it down)
