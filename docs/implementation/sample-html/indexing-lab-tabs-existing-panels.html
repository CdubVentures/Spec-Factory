Indexing Lab — Tabs over existing panels

Same panel internals as production (collapsible HOW IT WORKS, ALL IMAGES / SKUS, RUN HISTORY sections, tiny 3-letter chip-buttons CLR VIEW HERO LOOP EVAL). The only change is the outer shell: the four vertical accordions become four tabs so you only see one finder's panel at a time.

The tab bar is data-driven. A single FINDERS registry (see the script at the bottom of this file) is the source of truth. Adding a new finder tab = add one row to the array + drop one <div class="panel" data-finder="xyz"> in the HTML. See the How to add a tab card at the bottom.

How to add a new finder tab (2 steps)

  1. Add one row to the FINDERS registry in the script below.
  2. Drop one <div class="panel" data-finder="YOUR_ID">…</div> using any existing panel as a template. The tab bar picks it up automatically.
// ── add this line and you have a new tab ──────────────
const FINDERS = [
  { id: 'cef', name: 'Color & Edition', icon: '◈', kpi: '4c · 1ed',     status: 'complete' },
  { id: 'pif', name: 'Product Image',  icon: '▣', kpi: '0 / 44 carousel', status: 'empty'    },
  { id: 'rdf', name: 'Release Date',   icon: '◷', kpi: '3 / 4 resolved',  status: 'partial'  },
  { id: 'sku', name: 'SKU',            icon: '⌗', kpi: '4 / 4 published', status: 'complete' },

  // ↓ example: uncomment to add a "Materials" tab. That's the whole change. ↓
  // { id: 'materials', name: 'Materials', icon: '◆', kpi: '0 / 4 resolved', status: 'idle' },
];
