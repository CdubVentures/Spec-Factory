Views vs Heroes

  • top, bottom, left, right, front, rear, angle, sangle

  • Budget: 5 calls / view, target: 3 images

  • Full-scene 16:9 marketing images
  • Background kept (context matters)
  • Searched after all views done
  • Budget: 3 calls, target: 3 images

Learning (Per Variant)

Every call returns a discovery log (URLs checked + queries run). The next call — whether view or hero, within the same loop or a future run — receives the full accumulated history so it never repeats the same search.

Call 1 · View: Top
Searches corsair.com, bestbuy.com. Finds 2 images. Logs URLs + queries.
→
Call 2 · View: Left
Receives call 1's log — "don't repeat these." Tries amazon.com, newegg.com instead.
→
Call 5 · Hero
Receives calls 1–4's combined log. Finds lifestyle shots from entirely new sources.
📚
Images accumulate, not replace. Run 1 finds 3 images, Run 2 finds 3 more — you now have ~6 unique images. The discovery log also persists, so future runs pick up where you left off.

Evaluation & Carousel Slots

A vision LLM ranks candidates per view. Carousel slots auto-fill from the winner, but you can always override by dragging an image to a slot.

1. Manual Override Drag an image to a slot → always wins.
2. Eval Winner Vision LLM's top pick fills automatically.
3. Empty No images found yet for this slot.
