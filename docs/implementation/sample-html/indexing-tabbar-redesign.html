Indexing Lab — Tab Bar Redesign

Audit of the current tab strip and a self-contained nav-control replacement. The bar lives as its own rounded card — the active tab fills with the finder's color tint, the panel below sits on its own with a gap, and the brittle pseudo-dividers are gone.

target: IndexingTabBar.tsx · FinderTabBar.css

1Audit of the current tab strip IndexingTabBar.tsx · FinderTabBar.css

Working — preserve

  • ✓
    Semantic color tokens per finder. Each finder has iconClass keyed tokens (cef / pif / rdf / sku / key / pipeline) with matching fg/bg/border triplets. The redesign inherits these directly — no new color tokens.
  • ✓
    Keyboard navigation. finderTabKeyboard.ts: roving tabindex, Left/Right/Home/End. The redesign uses the same behavior — pure CSS/markup swap on the button.
  • ✓
    Cache-shared summary hooks. Each tab reuses the panel's queryKey. Keep the FinderTabSummary contract — { kpi, status } — unchanged. The redesign only changes how those two fields render.
  • ✓
    Pipeline-as-first-tab invariant. PIPELINE_TAB_ID is prepended to FINDER_PANELS in finderTabMeta.ts. Keep that ordering contract.

Issues — fix

  • ✗
    Active-state affordance is too quiet. Only .sf-tab-item-active's surface tint signals "selected." From 3 ft away you can't tell which tab is active. A filled finder-color tint reads from across the room.
  • ✗
    Divider pseudo-elements are a workaround. Absolute-positioned slivers hide around the active tab (FinderTabBar.css:27-30). A card with internal gap eliminates the patch entirely.
  • !
    KPI typography has no shared rhythm. Each finder invents its own shape: 18 runs · 56%, 4c · 1ed, 5 img · 4 var, 4 / 4 published, 6 / 80. The redesign standardizes one value · label strap and lets numeric magnitude carry the weight.
  • !
    Progress is hidden. Five of six tabs carry an implicit ratio. A 22 px progress ring in the finder's color communicates "3 of 4 resolved" in one glance — the status dot can't.
  • !
    Status dot is too small and ambiguous. 10 px + double ring fights the icon badge for attention, and idle vs empty are a muted circle vs a muted hollow ring — easy to miss.
  • !
    Running state is whisper-quiet. A pulsing dot. Live activity should be shouted — a dedicated Running pip with a colored breathe wins.
  • !
    Tab sizing is uneven across screen widths. min-width:180px flex-1 grows tabs unpredictably. Fine at 1200 px, ugly at 900 px. A dedicated narrow variant handles the fallback.

6Implementation notes contract stays, CSS gets a rewrite

What stays identical

  • Component signature. IndexingTabBar({ activeId, onSelect, productId, category }) — no prop changes.
  • Tab ordering. getIndexingTabIds() returns [PIPELINE_TAB_ID, ...FINDER_PANELS] — preserved.
  • Summary contract. FinderTabSummary = { kpi: string; status: FinderTabStatus } — preserved.
  • Hooks. useCefTabSummary, usePifTabSummary, etc. continue to reuse the panel's queryKey.
  • Keyboard nav. finderTabKeyboard.ts untouched.
  • Color tokens. .finder-tab-icon-<id> tokens remain the source of truth for per-finder color.

What changes

  • Markup. Tab row becomes a self-contained card — no more panel coupling. Each button renders [icon] [text stack] [status slot]. Inline SVG progress ring replaces the StatusDot when status is complete, partial, or empty and a numeric ratio is available.
  • CSS. FinderTabBar.css rewritten. Drops ::before dividers and the min-width:180 flex-1 scatter. Adds the tinted active fill + icon-mode-switch rules.
  • KPI shape. Summary hooks gain a structured variant — the current KPI string keeps rendering as a fallback, but the new shape exposes { value, total?, percent?, label } for ring rendering.
  • Status glyph rule. See table below — removes the idle / empty ambiguity in the old dot.
  • Layout on the page. IndexingPage renders the bar and the finder panel as siblings with a 12 px gap. Sticky positioning becomes available because nothing welds the two together.
