Indexing Lab — Phase 2 Polish

Starting point: the tabs are live and working. Looking at the current GUI there's still dead weight above the tabs and cramped cryptic rows below them. This sample pushes six moves:

  • Command Deck — one compact row replaces the big Run-IndexLab / Stop / Clear / Replay / force-kill stack above the tabs.
  • Product Chip — the three text inputs turn into a card-style chip: brand + model + [Change]. Inputs only appear in a "pick product" mode.
  • Bolder active tab — larger icon (38px), thicker accent border, 3px underline band connecting tab to panel.
  • Hero KPI — one dominant "here's how done you are" number per finder, with a progress bar. Secondary stats sit to the right, smaller.
  • Variant card (not row) — bigger rows with swatch + name + labeled progress bars + labeled icon-first action buttons. Status-coloured left border. 3-letter chip-buttons retired.
  • Progressive disclosure — HOW IT WORKS + RUN HISTORY folded by default; ALL VARIANTS is the action surface.

Summary — what this phase removes and what it adds

Current (after tabs shipped)

  • ~25% of viewport is consumed above the tabs by 5 full-width control elements (Run IndexLab blue, Stop red, Clear, Replay, force-kill checkbox).
  • Product picker still visually dominated by three text input boxes even after selection.
  • Tabs are thin — icon is 32px, accent ring is 1px, underline band is 2px. "Active" is communicated mostly by border-colour.
  • Four stat tiles are visually equal — the critical "0 / 44 carousel" number is siblings with Images / Variants / Runs.
  • Variant rows are dense — swatch + name + 2 chips + 5 tiny 3-letter buttons (CLR VIEW HERO LOOP EVAL) on one narrow line.
  • HOW IT WORKS and RUN HISTORY compete with the action area even when not needed.
  • Footer re-states "Last run · Model · Runs" already shown in the tab's KPI strip.

Phase 2 polish

  • Command Deck — single 56px strip: product chip + context pills + status pill + Replay + primary Run button + kebab. Stop only appears when running.
  • Product chip — brand accent badge + model headline + [▾ Change] opens the picker in a drawer.
  • Tabs grow — icon 38px with inset highlight, 1.5px accent ring, 3px underline band, shadow-lift on active.
  • Hero KPI — one dominant number with progress bar + caption answers "how done?". Secondary stats are 60% the size.
  • Variant cards — swatch + name + labeled progress bars + labeled icon-first buttons (Discover / Hero / Loop / Eval) + kebab for rarely-used actions. Status left-border (red / amber / green).
  • Progressive disclosure — HOW IT WORKS + RUN HISTORY collapsed by default. ALL VARIANTS is the surface you act on.
  • Footer removed — "Last run 12m ago" already lives in the Command Deck status pill.
