Manual Src Audit

Spec Factory `src/**` Audit

Scope Discipline

This report is only for src/**. Tools, scripts, docs, and GUI-react code are intentionally out of scope for this pass. Inventory and LOC totals were checked mechanically, but the dead/defer calls were revalidated manually from current file contents plus current import and route wiring evidence instead of relying on an autogenerated classifier.

Production vs Test Split

Method

  • Walked the full current `src` inventory and then rechecked every explicit dead/defer exception against current imports, runtime wiring, and README/domain notes.
  • Split the appendix and summary into production/support files versus test files so the test surface is visible separately from the non-test surface.
  • Marked only high-confidence residue as `DEAD_CANDIDATE`: the stale API contracts folder and the one-time catalog backfill file.
  • Kept documented but currently weak/test-only seams in `DEFER` instead of overstating dead-code certainty, including planned review prewiring that is not dead but not fully settled yet.
  • Treated tests, fixtures, audit notes, JSON manifests, and `.d.ts` files as `ACTIVE_SUPPORT` unless a stronger exact call existed.
