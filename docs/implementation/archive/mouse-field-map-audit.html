Informational (9)

I-01 lngth is a non-standard abbreviation of "length"
Likely to avoid JS reserved word. Aliases include "length" but the key itself is confusing. Consider length_mm or mouse_length.
I-02 connectivity vs connection semantic overlap
connection = mode (wired/wireless/hybrid). connectivity = technology (2.4GHz Dongle, etc.). Valid distinction but naming is confusing. Consider renaming connectivity to wireless_technology.
I-03 rgb field overlaps with lighting
rgb is typed string but appears boolean-like. lighting has structured zone values. Semantic boundary unclear. Consider making rgb a boolean.
I-04 design field is semantically undefined
No known_values, no meaningful tooltip. Unclear what it captures that shape, form_factor, and honeycomb_frame don't already cover.
I-05 Missing cross-validation rules for component consistency
Only 3 rules exist. Missing: sensor↔sensor_brand, switch↔switch_brand, encoder↔encoder_brand, wired→no battery, onboard_memory=yes→value required, bluetooth=yes→connection must include.
I-06 hump known_values naming inconsistency
Mixed separators: "back - front", "back - rear" use " - " while "back mid" uses space. Semantics are also ambiguous.
I-07 feet_material enum uses mixed casing
Values mix: "Glass", "PTFE", "PTFE (Black-Dyed)". normalize flag is lower_trim on the data_list, but compiled known_values retain original casing. Could cause case-sensitive matching failures.
I-08 form_factor values are handedness, not form factor
form_factor
Values: ambidextrous, left, right — this is handedness. But search_hints mention "small/medium/large" which is actual form factor/size. The field name misleads. Meanwhile shape has symmetrical, ergonomic, asymmetrical — which is closer to what "form factor" usually means.
I-09 price_range: string type but numeric sample values
price_range
Samples are "59.99", "99.99", "149.99" — actual prices, not bracket tokens. Parse notes say "bracket tokens like '0-50', '50-100'" but real data doesn't match. Should be type: "number", unit: "usd" or the sample data needs reformatting.
