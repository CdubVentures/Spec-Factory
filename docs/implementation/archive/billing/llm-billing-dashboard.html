Global billing_entries Schema in app.sqlite COMPLETE
billing_entries
  id            INTEGER PRIMARY KEY
  ts            TEXT        -- ISO timestamp
  month         TEXT        -- "YYYY-MM" (indexed)
  day           TEXT        -- "YYYY-MM-DD" (indexed)
  provider      TEXT        -- "openai", "anthropic", "google", "lab-openai"
  model         TEXT        -- "gpt-5", "claude-opus-4-6", "gemini-2.5-flash-lite"
  category      TEXT        -- "mouse", "monitor", "keyboard"
  product_id    TEXT        -- (indexed)
  run_id        TEXT        -- pipeline run correlation
  round         INTEGER     -- round within run
  prompt_tokens INTEGER
  completion_tokens INTEGER
  cached_prompt_tokens INTEGER
  total_tokens  INTEGER
  cost_usd      REAL
  reason        TEXT        -- "search_planner_enhance", "serp_url_selector", etc.
  estimated_usage INTEGER   -- 0/1 boolean
  meta          TEXT        -- JSON blob for extensibility

READY Schema lives in appDbSchema.js. Billing methods on AppDb class in appDb.js. All 10 call sources write via costLedger.appendCostLedgerEntry({ config, appDb, entry }). Dual-write: SQL to app.sqlite + JSONL to .workspace/global/billing/ledger/{month}.jsonl. Rebuild contract: on bootstrap, seedBillingFromJsonl() restores from JSONL if table is empty.

The existing GET /billing/{category}/monthly returns basic rollup. New endpoints add cross-category aggregation, time-series, and per-call drill-down.

  COUNT(*) as calls,

  SUM(prompt_tokens) as prompt_tokens,
  SUM(completion_tokens) as completion_tokens,
  SUM(cached_prompt_tokens) as cached_prompt_tokens,
  COUNT(DISTINCT model) as models_used,
  COUNT(DISTINCT category) as categories
FROM billing_entries
WHERE month = ?

6 Design Constraints

Zero-Drift UI

All colors via semantic tokens. No hardcoded hex in components. Chart colors from CALL_TYPE_REGISTRY only.

CQRS / Rebuild

billing_entries in app.sqlite is fully rebuildable from JSONL ledger. Delete DB → seedBillingFromJsonl() re-ingests .workspace/global/billing/ledger/*.jsonl on next bootstrap.

No New Deps

recharts, @tanstack/react-table, @tanstack/react-query all already installed. Zero new packages.

O(1) Extensibility

Adding a new LLM call source requires:

  1. Create adapter via createPhaseCallLlm (usage captured automatically)
  2. Add one row to CALL_TYPE_REGISTRY (label + color)

Dashboard, charts, filters, and DataTable all derive from the registry. Zero component changes.
