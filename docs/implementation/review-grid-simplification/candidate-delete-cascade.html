1. The Problem

Users need to delete individual candidates or all candidates for a field from the review grid. Today the only deletion path is via the finder panel (delete a CEF run), which doesn't cover pipeline sources or manual overrides. There is no review-grid-side deletion.

Deletion must propagate to all 4 data locations where candidate state lives, and must conditionally clean up backing artifacts (crawled pages, screenshots, finder runs) only when the last candidate from that source is removed.

Key invariant: Artifacts are shared across fields. A single CEF run produces candidates for both colors AND editions. A pipeline crawl of rtings.com may feed 10 fields. Never delete artifacts while other fields still reference them.

2. The 4 Data Locations

Location 1: SQL field_candidates table (specDb, per-category) One row per extraction event, keyed by source_id UNIQUE(category, product_id, field_key, source_id) Immutable after insert — only status changes (candidate → resolved) Location 2: product.json candidates[] .workspace/products/{pid}/product.json candidates.colors: [ { value, source_id, source_type, confidence, model, ... }, ... ] Flat entries, one per source — no sources[] accumulation Location 3: product.json fields[] .workspace/products/{pid}/product.json fields.colors: { value, confidence, source, sources[], linked_candidates[] } Derived state — must be republished from remaining candidates after delete Location 4: Finder / Pipeline Artifacts Custom finders (CEF/PIF): • color_edition.json / product_images.json (per-product JSON) • color_edition_finder + _runs SQL tables • Image files in product folder (PIF only) Pipeline (IndexLab): • runs SQL table + run_artifacts table • crawl_sources SQL table → .workspace/output/.../sources/{hash}.html • source_screenshots SQL table → .workspace/output/.../screenshots/{sid}.jpg • source_videos SQL table → .workspace/output/.../videos/{vid}.webm • .workspace/runs/{run_id}/ (runtime logs) • .workspace/output/{cat}/{pid}/runs/{rid}/ (all output files)

4. Source ID — The Universal Linking Key

source_id is the bridge between candidates and artifacts

source_id is stored on every field_candidates row and every product.json candidates[] entry. It is the only link back to the originating extraction event.

// Deterministic (finders) — survives rebuild
buildSourceId({ source: 'cef', run_number: 3 }, 'razer-viper')
  // → "cef-razer-viper-3"

// Unique (manual/review) — one per user action
buildSourceId({ source: 'manual_override' }, 'razer-viper')
  // → "manual-razer-viper-1712873400000"
Do NOT parse source_id to determine source type. The format {source}-{pid}-{runNumber} is ambiguous because product IDs contain hyphens (e.g., cef-razer-viper-v3-pro-1). Instead, read the source_type column directly from the field_candidates row — it is already populated on every row. For run_number extraction, use lastIndexOf('-') on source_id (the last segment is always numeric for finders).
Pipeline linking gap: field_candidates stores source_id but not run_id. The pipeline runs table uses run_id (string like idx-2026-04-10-mouse-1) and has no run_number column. For pipeline artifact cleanup, either: (a) store run_id in metadata_json at candidate submission time, or (b) add a run_id column to field_candidates. This must be solved when pipeline feeds the grid. Finders (CEF/PIF) are unaffected — they use run_number which maps directly to their runs tables.

6. Delete Contract

Action A: Delete Single Candidate

Trigger: User clicks delete button on a candidate card in the review grid drawer

Confirmation: "Delete this candidate? This will remove the [source_type] extraction for [field_key]. If this is the last candidate from this source, the backing run/artifacts will also be removed."

Input: category, productId, fieldKey, sourceId

Steps:

  1. Delete field_candidates row by (productId, fieldKey, sourceId)
  2. Remove matching entry from product.json candidates[fieldKey]
  3. Republish product.json fields[fieldKey] from remaining candidates (confidence gate + set_union merge)
  4. Check: SELECT COUNT(*) FROM field_candidates WHERE product_id=? AND source_id=? (across ALL fields)
  5. If count = 0 (last candidate from this source): cascade to artifact cleanup
  6. Emit data-change event for GUI invalidation

Action B: Delete All Candidates for a Field

Trigger: User clicks "Delete All" button below "Review All" for a field

Confirmation: "Delete ALL candidates for [field_key]? This will remove [N] candidates from [M] sources. Sources with no remaining candidates in other fields will have their artifacts removed."

Input: category, productId, fieldKey

Steps:

  1. Collect all source_ids for this field: SELECT DISTINCT source_id FROM field_candidates WHERE product_id=? AND field_key=?
  2. Delete all rows: DELETE FROM field_candidates WHERE product_id=? AND field_key=?
  3. Remove product.json candidates[fieldKey] entirely
  4. Remove product.json fields[fieldKey] (unpublish)
  5. For each collected source_id: check cross-field count, cascade if zero
  6. Emit data-change event

Artifact Cascade (Step 5 detail)

Only runs when zero candidates remain for a source_id across ALL fields.

Source TypeCascade Actions
cef Delete run from color_edition_finder_runs (by run_number)
Remove run entry from color_edition.json runs[]
Update color_edition_finder summary (run_count, selected)
If all runs gone: delete summary row + JSON file
pif Same as CEF + delete associated image files in product folder
pipeline Call existing deletionStore.deleteRun({ runId, productId, category, fsRoots }) which handles:
• SQL: runs, run_artifacts, crawl_sources, source_screenshots, source_videos
• product.json: filter runs[] entries
• Filesystem: rmSync(.workspace/runs/{rid}) + rmSync(.workspace/output/.../runs/{rid})
manual_override / review No artifacts — cascade is a no-op

7. Cascade Logic (Pseudocode)

async function deleteCandidateBySourceId({ specDb, category, productId, fieldKey, sourceId, config, productRoot }) {

  // --- Location 1: SQL ---
  specDb.deleteFieldCandidateBySourceId(productId, fieldKey, sourceId);

  // --- Location 2: product.json candidates[] ---
  const productJson = readProductJson(productRoot, productId);
  if (productJson?.candidates?.[fieldKey]) {
    productJson.candidates[fieldKey] = productJson.candidates[fieldKey]
      .filter(e => e.source_id !== sourceId);
  }

  // --- Location 3: product.json fields[] (republish) ---
  const remaining = specDb.getFieldCandidatesByProductAndField(productId, fieldKey);
  if (remaining.length === 0) {
    delete productJson.fields?.[fieldKey];  // unpublish
    specDb.demoteResolvedCandidates(productId, fieldKey);
  } else {
    // Reuse republishAfterDelete logic:
    // - confidence gate (config.publishConfidenceThreshold ?? 0.7)
    // - set_union merge if applicable
    // - scalar: pick highest-confidence above threshold
    republishField(specDb, productId, fieldKey, remaining, config, productJson);
  }

  writeProductJson(productRoot, productId, productJson);

  // --- Location 4: Artifact cascade (conditional) ---
  const crossFieldCount = specDb.countCandidatesBySourceId(productId, sourceId);
  if (crossFieldCount === 0) {
    await cascadeArtifactDelete(specDb, productId, sourceId, productRoot);
  }

  emitDataChange({ event: 'candidate-deleted', category, productId, fieldKey });
}


// WHY: Do NOT parse source_id to get source_type — it's ambiguous (product IDs have hyphens).
// Instead, pass source_type from the field_candidates row (already a column on every row).
// For run_number: safe to extract via lastIndexOf('-') — last segment is always numeric for finders.

async function cascadeArtifactDelete(specDb, category, productId, sourceId, sourceType, productRoot) {

  // Extract run_number from last segment of source_id (safe: always numeric for finders)
  const lastDash = sourceId.lastIndexOf('-');
  const lastSeg = lastDash >= 0 ? sourceId.slice(lastDash + 1) : '';
  const runNumber = /^\d+$/.test(lastSeg) ? Number(lastSeg) : null;

  if (sourceType === 'cef' && runNumber != null) {
    // Actual method: finderSqlStore.removeRun(productId, runNumber)
    cefSqlStore.removeRun(productId, runNumber);
    cefJsonStore.deleteRun({ productId, runNumber });
    // Recalculate summary from remaining runs
    updateFinderSummary(specDb, productId);
  }

  if (sourceType === 'pif' && runNumber != null) {
    pifSqlStore.removeRun(productId, runNumber);
    pifJsonStore.deleteRun({ productId, runNumber });
    deleteOrphanedImageFiles(productRoot, productId);
    updateFinderSummary(specDb, productId);
  }

  if (sourceType === 'pipeline') {
    // Pipeline runs table has NO run_number column — uses run_id (string).
    // run_id must be stored in metadata_json at candidate submission time,
    // OR a run_id column must be added to field_candidates. (Future work.)
    const runId = resolveRunIdFromMetadata(specDb, productId, sourceId);
    if (runId) {
      // Actual signature: deleteRun({ runId, productId, category, fsRoots })
      await deletionStore.deleteRun({
        runId,
        productId,
        category,
        fsRoots: { runsRoot: '.workspace/runs', outputRoot: '.workspace/output' },
      });
    }
  }

  // manual_override / review: no-op (no artifacts)
}
