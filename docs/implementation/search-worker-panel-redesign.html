v3 — Integrated Query/Response Design
Single-flow card per attempt — No more awkward 3-column split. Each expanded attempt flows top-to-bottom: query header → triage bar → result list. The query IS the card header, the Serper response IS the body.
Triage visualization bar — At-a-glance ratio of keep/maybe/drop/hard_drop with colored segments + counts below. Shows immediately how useful the query was.
Clean result rows — 6-column grid: rank, fetched dot, decision chip, score, domain URL, fetch worker tag. Dropped results are dimmed (opacity 0.5). Fetched results show their exact/host linkage and fetch worker ID.
Collapsed summary is complete — Each row shows: attempt #, query text, provider pill, status, result count, duration. You can scan all queries without expanding.
State indicators — Running rows have left accent border. Zero-result rows have left warning border. Fallback attempts show "f1" in warning color. All immediately scannable.
Data source — All from WorkerDetailResponse.search_history[]. Each SearchWorkerAttempt has query, provider, status, result_count, duration_ms, and results[] array with decision/score/fetched/domain/fetch_worker_id.
