v3 Hero Section Changes
HeroBand container — Single elevated surface with border, matching every prefetch/fetch stage panel. Title row uses same pattern: 26px bold title + italic subtitle (left) · trailing chips (right).
HeroStatGrid (6-col) — Large text-4xl monospace values with tiny uppercase labels below. Same visual weight as NeedSet/SearchPlanner/CookieConsent panels. Color-coded by semantic meaning (accent=total, success=completed, warning=cost, teal=latency).
Narrative line — Italic summary sentence with bold highlights, matching how PrefetchNeedSetPanel and PrefetchSearchPlannerPanel present their context. Auto-generated from summary stats.
Visuals row — Three visual cards: (1) Token Split with per-model breakdown bars, (2) SVG donut chart for model distribution replacing the flat list, (3) Cost by Call Type horizontal bars. All use the same visual-card container style.
Footer strip — Matches HeroBand footer pattern (10px uppercase, border-top separator, secondary metrics). Houses low-priority stats that previously cluttered the header.
Removed — The old custom header bar with scattered chips. The old MiniStat cards. The old collapsible toggle wrapper (hero is always visible; call log scrolls independently).
