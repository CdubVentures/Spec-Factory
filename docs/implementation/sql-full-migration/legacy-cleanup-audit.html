Bug 1: Double-Nesting Path in LocalStorage

File: src/s3/storage.js:206-214 — resolveLocalPath()
Evidence: %LOCALAPPDATA%\spec-factory\output\specs\outputs\_billing\ exists on disk (prefix duplicated)

resolveOutputKey() returns keys like specs/outputs/_billing/monthly/2026-03.txt (with S3 prefix baked in). resolveLocalPath() then joins the full key onto outputRoot, producing output/specs/outputs/_billing/.... The prefix specs/outputs appears twice.

// storage.js:206-214 — the bug
resolveLocalPath(key) {
  if (key.startsWith(`${this.outputPrefix}/`)) {
    return path.join(this.outputRoot, ...key.split('/'));  // <— full key includes prefix
  }
  return path.join(this.outputRoot, ...key.split('/'));
}
// Fix: strip prefix before join
// return path.join(this.outputRoot, ...key.slice(this.outputPrefix.length + 1).split('/'));

Root cause: src/shared/storageKeyPrefixes.js defines OUTPUT_KEY_PREFIX = 'specs/outputs'. This is correct for S3 but should be stripped for local filesystem paths.
