1. High-Level Pipeline Overview

Scope of this document: Everything that happens from when the Domain Classifier produces the ordered fetch plan through Crawlee execution, plugin hooks, extraction (screenshot capture, video recording), artifact persistence to disk and database, and the transform phase that runs after page closure.
DOMAIN CLASSIFIER ——> CRAWLEE BATCH LOOP ——> NAVIGATION ——> PLUGIN HOOKS
                                                                        ↓
ARTIFACT PERSIST ←—— EXTRACTION (Screenshot/Video) ←—— PAGE CAPTURE ←—— BLOCK DETECT
     ↓
RESOLVE + EVENTS ——> TRANSFORM PHASE (no page, no timeout pressure) ——> NEXT BATCH

3. Step-by-Step Flow Diagram

A Domain Classifier — Build Ordered Fetch Plan runDomainClassifier.js

Transforms discovery results into an executable crawl plan. Runs synchronously before any browser work begins.

  • Merge sources: discovery-approved URLs + seed URLs + learning seed URLs
  • Filter out blocked hosts (runtime overrides from operator)
  • Sort all URLs by SERP slot rank (highest search relevance first)
  • Apply URL cap via domainClassifierUrlCap setting
  • Assign sequential worker IDs: fetch-1, fetch-2, fetch-3, ...
  • Emit source_fetch_queued per URL with triage metadata

Output: { orderedSources[], workerIdMap, stats }

Each source carries: url, host, discoveredFrom (seed | learning_seed | discovery_approved), triageMeta (triage_score, search_slot, search_rank, identity_prelim, host_trust_class), workerId

↓orderedSources + workerIdMap
B Batch Loop & Crawlee Dispatch crawlSession.js:runFetchPlan()

URLs are processed in batches through Crawlee's AutoscaledPool. Each batch runs up to slotCount concurrent browser pages.

  • Batch size = slotCount × 2 (default: 4 slots × 2 = 8 URLs per batch)
  • Each URL gets a unique key: url::timestamp::random8 → mapped to a Promise
  • Calls crawler.run(requests) — Crawlee dequeues from its internal AutoscaledPool
  • Global timeout check at start of each batch: if (Date.now() - startMs ≥ maxRunMs) break
  • After batch: block detection → proxy retry for blocked URLs → transform phase → next batch
Concurrency Architecture

crawlMaxConcurrentSlots4Parallel browser pages (min & max concurrency)

↓per URL (parallel, up to slotCount)
Handler Timeout Budget: Everything below (steps C through J) must complete within the handler timeout. Default: 45 seconds total. If exceeded, the timeout rescue mechanism activates — see Error Paths below.
C Pre-Navigation Hooks crawlSession.js:preNavigationHooks[]

Runs before page.goto(). Sets up anti-bot delay, live streaming, and fires onInit plugin hook so addInitScript() and page.route() take effect on initial page load.

  • Same-domain delay: 2s ± 50% jitter between requests to same host (anti-bot)
  • Start CDP Screencast (JPEG, quality 30, 1280×720, every 3rd frame)
  • onInit hook — plugins that must run before navigation:
    • stealth — addInitScript(): fingerprint spoofing, webdriver flag hide
    • overlayDismissal — addInitScript(): CSS suppression for popups + MutationObserver for delayed overlays
    • cssOverride — page.route(): block CSS domain requests
↓screencast streaming, delay applied
D Page Navigation (Crawlee Native) Crawlee → page.goto(url, gotoOptions)

Crawlee handles navigation natively via navigationTimeoutSecs. Playwright uses default waitUntil: 'load'.

  • page.goto(url, { timeout: navigationTimeoutSecs * 1000 }) — set by Crawlee internally
  • Waits for load event (DOM + images + fonts + stylesheets)
  • Navigation timeout: 20s (configurable via crawleeNavigationTimeoutSecs)
  • On timeout: errorHandler marks as non-retryable (page genuinely unresponsive)
  • Total Crawlee budget: nav(20) + handler(45) + buffer(10) = 75s
↓load event fired (page fully loaded)
F Fetch Suite Loop — Dismiss / Scroll / Dismiss suiteOrchestrator.js → pluginRunner.js

Round-based orchestrator replaces the old sequential hook model. Six plugins across 3 hook types, with configurable rounds.

F.1 Loading delay fetchLoadingDelayMs: 3s — wait for JS hydration before first dismiss
F.2 onDismiss (initial cleanup) — sequential or concurrent per fetchSuiteMode
F.3 Rounds 1..N (fetchDismissRounds, default 2):
       onScroll → onDismiss (catches scroll-triggered popups)

Plugin × Hook Matrix

PluginonInitonDismissonScrollTimeoutWhat It Does
stealth ✓—— immediate addInitScript(): fingerprint spoof, webdriver hide
cookieConsent —✓— 5s CMP detection + banner dismissal (Shopify, OneTrust, etc.)
overlayDismissal ✓✓— 800ms settle Layer 1: CSS suppression + MutationObserver (onInit). Layer 2: DOM scan + close-click. Layer 3: scroll-lock reset (onDismiss).

Suite lifecycle: [3s loading delay] → onDismiss → (onScroll → onDismiss) × 2 rounds → capture. Total dismiss-suite time: ~8-25s depending on content. The domExpansion budget (15s) prevents starvation of downstream extraction.
↓page interactive, overlays dismissed, content expanded
G Page Capture & Block Detection crawlSession.js:requestHandler L122-155

Capture the page's current state and check for bot blocks before proceeding to extraction.

G.1 html = await page.content() — full DOM serialization
G.2 finalUrl = page.url() — may differ from original (redirects)
G.3 title = await page.title()
G.4 status = response.status() (or 200 if unavailable)
G.5 Stash rescue anchor: request.userData.__capturedPage = { html, finalUrl, title, status }
G.6 Block detection: classifyBlockStatus({ status, html })

↓
H Extraction Plugins — Capture Phase extractionRunner.js:runCaptures()

Two-phase extraction runner. Capture phase runs inside the handler with live page access, under timeout pressure.

  • Sequential plugins run first (for-of await loop) — may mutate page state
  • Concurrent plugins run after via Promise.all() — read-only operations
  • All contexts frozen via Object.freeze() to prevent cross-plugin mutation
  • Per-plugin try/catch — one failure doesn't affect others

Screenshot Plugin (phase: capture, concurrent: true)

H.1 Check enabled: settings.capturePageScreenshotEnabled
       If false → return { screenshots: [] } immediately
H.2 Page Stabilization 1,500ms timeout
       Single page.evaluate() runs all gates in parallel inside browser:
  • document.fonts.ready — all web fonts loaded
  • HTMLImageElement.decode() — all images decoded
  • Double requestAnimationFrame — paint cycle complete ~33ms
  • Measure scrollHeight vs viewportHeight — detect tall pages
       Returns: { stabilized, durationMs, gates, scrollHeight, viewportHeight, exceedsLimit }
       Never throws — returns { stabilized: false } on timeout
H.3 Full-page screenshot (captured first — priority)
       page.screenshot({ fullPage: true, type: 'jpeg', quality: 75 })
       Max file size: 5 MB. Format/quality configurable.
H.4 Selector crops (optional, max 12 selectors)
       Parse comma-separated selectors from capturePageScreenshotSelectors
       Per selector: page.$(sel) → element.screenshot()
       Failures swallowed per-selector (one bad selector doesn't stop others)
H.5 Scroll-and-stitch (if page exceeds 16,384px Chromium limit)
       Requires sharp library (optional dependency, skip if unavailable)
       Scroll viewport-by-viewport, capture each section
       Double-rAF between scrolls ~33ms/section to ensure paint
       Composite all sections → single tall image, mark stitched: true
       Reset scroll position to 0 after stitching

Video Recording (Playwright built-in, not extraction plugin)

V.1 Setup: prePageCreateHook sets pageOptions.recordVideo = { dir, size }
       Directory: $TMPDIR/spec-factory-crawl-videos/{runId}/
       Default size: 1280×720 (configurable via crawlVideoRecordingSize)
       Enabled only when crawlVideoRecordingEnabled === true
V.2 Recording: Playwright records automatically during entire page lifecycle
       From page creation through navigation, plugins, extraction
V.3 Save: prePageCloseHook calls video.saveAs(videoDir/{workerId}.webm)
       Async, fire-and-forget — pushed to pendingVideoSaves[]
       Awaited after batch completes via Promise.allSettled(pendingVideoSaves)
Timeout rescue for extraction: After extraction completes, results are stashed on request.userData.__capturedExtractions. If the handler times out after this point but before resolveEntry(), the failedRequestHandler recovers these extractions with timeoutRescued: true.
↓screenshots[] + extraction results
I Artifact Persistence (Disk + Database) screenshotArtifactPersister.js

Two-stage persistence: disk write (always) + SQL index (best-effort).

I.1 Disk write: {runDir}/screenshots/screenshot-{workerId}-{urlHash8}-{index:02d}-{kind}.{ext}
       SHA256 first 8 chars for content-addressed filenames
       Per-file try/catch — one failure doesn't stop others
I.2 SQL index: Insert to source_screenshots table
       Fields: screenshot_id (SHA256), content_hash, category, product_id, run_id,
       source_url, host, selector, format, width, height, size_bytes, file_path, captured_at
       Injected via DI callback insertScreenshot(row) — SQL failures don't block pipeline
I.3 Events: visual_asset_captured extraction_artifacts_persisted
↓
I+ Final Screencast Frame → GUI crawlSession.js L181-199

Emit latest screenshot as screencast frame for live GUI worker detail view.

  • Find last kind === 'page' screenshot from extraction results
  • Convert to base64, emit via onScreencastFrame({ worker_id, data, width, height, ts })
↓
J Resolution & Event Emission crawlSession.js L240-275
  • Stop CDP screencast for this page
  • Emit source_processed (marks fetch complete in GUI, increments worker doc count)
  • resolveEntry(uniqueKey, result) → resolves the processUrl promise
  • Crawlee closes the page → prePageCloseHook saves video (if recording enabled)
↓all URLs in batch resolved
K Transform Phase (Post-Page, No Timeout Pressure) extractionRunner.js:runTransforms()

Runs after all pages in the batch close. No page object available. No handler timeout.

  • Always concurrent via Promise.all() (ignores concurrent flag)
  • Context: { html, finalUrl, title, status, settings, captures }
  • Use case: HTML parsing, image compression, metadata extraction
  • Results merged into result.extractions
  • Errors logged but don't fail the URL

After transform: await Promise.allSettled(pendingVideoSaves) → consolidate video files

Emit crawler_stats (status codes, retry histogram, avg ms per request)

→ Next batch (or exit if all URLs processed / global timeout exceeded)

↓all batches complete
END Run Complete runProduct.js
  • Return { crawlResults } to pipeline orchestrator
  • Emit run_completed with urls_crawled, urls_successful, urls_blocked, duration_ms
  • session.shutdown() — close Crawlee crawler + proxy crawler

Error Paths — Timeout Rescue & Retry

E1 errorHandler (before each retry) crawlSession.js L339-377

Fires when requestHandler throws. Decides whether retry is worthwhile.

ErrorActionReason
ERR_NAME_NOT_RESOLVEDnoRetry = trueDomain doesn't exist

ERR_TUNNEL_CONNECTION_FAILEDnoRetry = trueProxy tunnel broken

Other errorsRetry (session rotation)Fresh fingerprint may help

↓retries exhausted or noRetry
E2 failedRequestHandler (final failure) crawlSession.js L278-334

Executes when request exhausts retries or errorHandler sets noRetry = true.

1 Block detected? Return block info with HTML/status
2 Page captured before timeout? Rescue HTML + screenshots from __capturedPage / __capturedExtractions
      Return partial result with timeoutRescued: true
      GUI shows "Partial" (yellow) instead of "Failed" (red)
3 No data captured? Return empty result with fetchError

Emit source_fetch_failed with timeout_rescued flag

E3 Two-Pass Proxy Retry crawlSession.js:retryWithProxy()

After batch completes, blocked URLs (except robots_blocked) are retried through a dedicated proxy crawler.

  • Separate PlaywrightCrawler instance with ProxyConfiguration
  • Max 2 retries on proxy crawler
  • Results replace blocked results from main batch

Failure Path — Block Detected (403/429/CAPTCHA)

Page loads but block detected by classifyBlockStatus(). Session retired, error thrown. Crawlee retries with fresh session (1 retry). If still blocked, failedRequestHandler resolves with block info. Blocked URLs later retried via proxy crawler.
