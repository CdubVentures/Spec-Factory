Spec Factory `tools/**` Audit

Scope Discipline

This report is only for tools/**. The audited inventory excludes the installed package tree under tools/gui-react/node_modules/**, but keeps handwritten tooling, checked-in GUI code, launchers, sidecars, build outputs, generated baselines, and sample outputs that still live in the repo. The inventory files only guarantee every in-scope current path is present; the status calls and exceptions below are written by hand in this report.

Method

  • Separated active handwritten tooling from build outputs, generated type files, sample validation output, and the excluded tools/gui-react/node_modules/** package install.
  • Protected the disabled review-grid and component-review surfaces as FUTURE_PREWIRED, while keeping Review LLM marked active because its route and taxonomy helpers are still live.
  • Marked only high-confidence dead GUI residue where files exist in the tree but have no current imports or route registration.
  • Counted LOC as physical lines for in-scope text-like files; the two .ico launcher assets are kept in file counts but do not contribute LOC.
  • Treat the primary LOC number as live production tool code: GUI runtime source plus launcher/runtime entrypoints, excluding dist/build output, generated baselines, tests, docs, configs/assets, hidden test-mode support, and dead-candidate residue.
  • Kept tests, docs, launch assets, config files, and helper seams as support unless there was stronger dead or future evidence.
