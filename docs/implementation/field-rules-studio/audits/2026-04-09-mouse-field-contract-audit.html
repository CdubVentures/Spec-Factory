Executive Summary

The mouse category field studio map defines 85 fields across 10 groups, backed by 4 component databases (116 sensors, 106 switches, 40 encoders, 3 materials) and 15 enum definitions. While the overall structure is solid, the audit identified several categories of issues:

  • Critical: Garbage/test values polluting production enums; corrupt component data; 100% missing component properties; canonical key typo
  • High: Missing range constraints on numeric fields; numeric fields without units; low component DB coverage; redundant unk tokens in enums
  • Medium: 19 boolean fields with wrong input controls; generic template tooltips; missing aliases; group casing inconsistency
  • Low: All ai_assist configs null; missing newer sensors/MCUs; tooltip source inconsistencies

Critical Issues (Must Fix)

CRITICAL-1 Garbage Values in Production Enums

Known values contain obvious test/garbage data that will corrupt validation and LLM extraction:

EnumGarbage ValuesImpact
coating"A", "help"LLM may match real coating data to these garbage tokens
feet_material"l"Single-char typo will match partial strings
switch_type"eqwe", "yay"Pollutes switch type vocabulary; alias matching may false-match
connectivity"1", "2.5"Ambiguous values with no clear meaning in connectivity context
CRITICAL-2 Corrupt Component Data: Encoder

pulsar Blue Encoder has encoder_steps: "hellow" — a garbage string that cannot be coerced to a number. This will cause type coercion failures for any product using this encoder.

CRITICAL-3 100% Missing Component Properties

Two component properties are defined in the schema but have zero data across all entities:

PropertyComponentCoverageImpact
releasing_forceswitches (106 entities)0%Cross-field validation impossible; field always returns unk
encoder_life_spanencoders (40 entities)0%Component-derived field produces no data
CRITICAL-4 Canonical Key Typo: "lngth"

The field key lngth is a typo of length. The key migration system reversed this (length → lngth) because "length" is not a generated field key. The typo is now canonicalized. LLM extraction will look for "length" — aliases must cover this. The cross-validation rule dimensions_consistency references lngth.
