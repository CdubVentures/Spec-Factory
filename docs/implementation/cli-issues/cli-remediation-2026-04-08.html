3. Known remaining issues

run_profile hardcoding — open bug

spec.js:294 hardcodes output.run_profile = 'standard' regardless of --profile or --thorough CLI flags. The flags are parsed by args.js but never applied to the output. One-liner fix.

filterKeysByBrand reads JSON redundantly — optimization

discoverCommand.js calls specDb.getAllProducts() (returns brand), then passes product IDs to filterKeysByBrand which re-reads each product JSON from storage to check brand. Should filter from the already-fetched DB rows.

In-process compile/validate — deferred

Converting compile/validate from child-process spawning to in-process function calls. The primary value: compile is currently blocked while indexlab runs (they share the global process slot). Three review rounds revealed this requires decoupling compile status from the shared processStatus global state, which powers the app shell, sidebar, and runtime ops UI. This is a state architecture change, not a CLI cleanup — deferred to a dedicated design session.

Blocking issues identified:

  • StudioPageShell.tsx:207 disables compile when processStatus.running is true (any process, not just compile)
  • setProcessStatus() is shared global state — a compile response { running: false } would corrupt an active indexlab run's status
  • Per-category concurrency lock needed (compile writes to _generated/, validate reads from it)
  • Frontend mutation contract tests expect { running: true } process snapshot responses
Rebuild contract for migrate-to-sqlite phases 2 & 4 — deferred

Phase 2 imports billing JSONL → SQL. Phase 4 imports learning profiles JSON → SQL. No reverse path exists. If spec.sqlite is deleted, billing entries and learning profiles are lost. Fix requires schema change + source-key tracking.

pipelineCommands.js still 352 LOC — deferred

commandIndexLab is the sole remaining command but handles: seed parsing, identity resolution, bridge setup, IPC forwarding, config overrides, checkpoint writing, run summary serialization, and bridge finalization. Under the 700 LOC soft limit. Decomposition is optional — only valuable if IndexLab orchestration changes frequently.
