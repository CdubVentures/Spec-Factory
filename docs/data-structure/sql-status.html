Performance Notes

Architecture

Design decisions that ensure the SQL layer performs well under pipeline workloads.

WAL Mode

SQLite Write-Ahead Logging enabled on all database surfaces. Allows concurrent reads during writes without blocking the pipeline.

Content-Addressed Dedup

Content hashing (content_hash) provides deterministic deduplication across evidence and source artifacts.

Optimized Indexes

Indexes tuned for pipeline query patterns: product+field lookups, run-scoped queries, source domain filtering, and component type searches.

Event Lifecycle

bridge_events can be purged after run finalize. All data is preserved in run_summary artifacts before cleanup.

Per-Category Isolation

SpecDb is one SQLite file per category. No cross-category lock contention. Categories can be operated on independently.

Database Surface Architecture

Two distinct SQLite database surfaces, each serving a specific domain concern.
