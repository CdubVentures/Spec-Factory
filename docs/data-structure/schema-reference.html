SpecDb

Per-Category 55 tables (54 + 1 FTS)

One SQLite database per category (.workspace/db/{category}/spec.sqlite). 55 tables from the base schema (54 regular + 1 FTS virtual). Additional runtime-created tables (learning stores, crawl_media) may exist in production databases. Holds all domain data: products, candidates, components, reviews, evidence, billing, telemetry, and source intelligence.

Runtime-Created (Not in Base Schema)

Learning stores (component_lexicon, field_anchors, url_memory, domain_field_yield) are created by learningStores.js on any category that runs through the pipeline. They only appear in mouse currently because mouse has the most pipeline runs — there is no category-specific logic. Ghost tables (crawl_media, domain_classifications, metrics) have no CREATE TABLE in the current source code — they are orphaned artifacts from deleted code that persist because SQLite does not auto-drop tables.
