1 Architecture: Three Independent Feeds

i
All three systems discover URLs, domains, and queries independently. They do not share discovery data with each other. Each maintains its own storage, accumulation, and dedup logic.

Discovery Independence Model

Indexing Pipeline
Search engines (Google, Bing, Brave, Searxng, Serper) → URL corpus → Domain authority index → Crawl & extract specs
SQL-backed Cooldown TTL
Color Edition Finder
LLM web browsing → Manufacturer + retailer pages → Discover colors, editions, variants
JSON + SQL No URL TTL
Product Image Finder
LLM web search → Per-variant queries → Download, dedup, quality-gate images
JSON + SQL No URL TTL
No data flows between the three feeds. Each discovers, stores, and accumulates independently.

4 Domain Tracking Flow

PIPELINE Rich Domain Model

sourceRegistry provides tiered authority:

TierRole
tier1Manufacturer (official)
tier2Lab / instrumented review
tier3Retailer (Amazon, Best Buy)
tier4Community / forum
tier5Aggregator / comparison

Each entry tracks: crawl config (rate limits, method), discovery config (priority, enabled), health stats (success rate, latency, block rate).

urlClassifier enriches each discovered URL with: tier, role, doc_kind_guess, identity_match_level, approvedDomain.

Crawl ledger aggregates domain stats: fetch_count, ok_count, blocked_count, avg_latency, cooldown_remaining.

CEF No Domain Model
!
No domain-level tracking exists.
URLs are stored as flat string arrays. No host extraction. No domain aggregation. No tier classification.

The LLM is asked to check "manufacturer site, major retailers, press releases" — but there's no structured record of which domains were checked.
PIF Minimal Domain Model
!
Per-image source_page only.
Each image record has a source_page field (full URL string where the image was found). No host extraction, no domain-level aggregation or bucketing.

Image download URLs are normalized (Best Buy, Shopify) but source_page is stored raw.

  fields_filled, fetch_success, ts, created_at

-- Live state: per-URL per-product fetch history
url_crawl_ledger (
  canonical_url, product_id,  -- PK (composite)
  category, original_url, domain, path_sig, final_url,
  content_hash, content_type, http_status, bytes, elapsed_ms,

  server_error_count, redirect_count, notfound_count, gone_count,
  first_seen_ts, last_seen_ts, first_seen_run_id, last_seen_run_id

-- Telemetry: every query executed during discovery runs
query_index (
  id, category, run_id, product_id, query, provider,
  result_count, field_yield, tier, ts, created_at
)  -- Indexes: category, run_id, (query, provider)

query_cooldowns (
  query_hash, product_id,  -- PK (composite)
  category, query_text, provider, tier, group_key,
  normalized_key, hint_source, attempt_count, result_count,
  last_executed_at, cooldown_until  -- THE ONLY TTL COLUMN
)  -- Indexes: product_id, cooldown_until

-- Content-addressed binary index (HTML, screenshots, PDFs)
crawl_sources (
  content_hash, product_id,  -- PK (composite)
  category, run_id, source_url, final_url, host,
  http_status, doc_kind, source_tier, content_type,
  size_bytes, file_path, has_screenshot, has_pdf,
  has_ldjson, has_dom_snippet, crawled_at
)  -- Indexes: product_id, run_id, host

8 Uniqueness & Deduplication

PIPELINE 3-Layer Dedup

Layer 1: Query Hash

stableHashString(productId + "::" + normalizedQuery) → PK in query_cooldowns. Prevents re-executing within TTL window.

Layer 2: Canonical URL

canonicalizeUrl() → PK in url_crawl_ledger. Same canonical URL + product_id = upsert (updates fetch counts).

Layer 3: Content Hash

SHA-256 of fetched content → PK in crawl_sources. Same content from different URLs = single record.

CEF 1-Layer Dedup

Layer 1: Set-Based String Match

JS new Set() during accumulateUrlsChecked(). Exact string equality only.

!
No normalization means:
https://corsair.com/k100 and
https://corsair.com/K100
are treated as different URLs.
PIF 2-Layer Dedup

Layer 1: Normalized URL Set

normalizeImageUrl(url) then check alreadyDownloadedUrls Set. Prevents re-downloading same image URL for a variant.

Layer 2: Content Hash

SHA-256 of downloaded file bytes. alreadyDownloadedHashes Set catches CDN re-serves (same image, different URL).

Self-heals from disk: if sets are empty at init, reconstructs from existing JSON image records.

13 Recommendations

i
The three feeds being independent is correct by design. The maintenance gaps are around unbounded accumulation, missing normalization, and zero URL health awareness in the finders.

PIF URL Accumulation Cap

Add a configurable max_urls_injected knob to PIF. When accumulated count exceeds cap, inject only the most recent N (FIFO). CEF resolved — URLs no longer injected into CEF prompts.

MEDIUM PIF only

Domain Extraction for PIF

Extract host from source_page at storage time and store as a separate source_domain field. Enables domain-level aggregation queries in the GUI without URL parsing at read time.
