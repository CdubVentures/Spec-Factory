Product Image Finder (PIF) Reference

Downloads high-resolution product images per color variant and edition via LLM web search

1 Overview
CEF-Dependent Image Discovery

The Product Image Finder discovers and downloads product images for each colorway defined by the Color & Edition Finder (CEF). It uses a web-search LLM to locate high-resolution product photos, firing all variant searches concurrently with a 1-second stagger between launches.

2 Variant Resolution
Search Target Resolution from CEF Data

PIF builds its search list from CEF's colors array. Each entry becomes a search target. The search label is resolved using a strict priority chain — never searching raw atoms when a better name exists.

  • Priority 1: Edition display name — If a combo entry matches an edition, search by edition name.
    Example: "Call of Duty Black Ops 6 Edition" instead of the underlying color atom.
  • Priority 2: Marketing color name — If a color has a color_name, search by that.
    Example: "Frost White" not "ivory".
  • Priority 3: Raw atom (fallback) — If neither edition nor color_name exists, search by atom.
    Example: "black".
Source
CEF colors array — each entry is one search target

3 Identity

5 Multi-Image Search
Candidate Ranking and Gallery Tagging

The search prompt asks the LLM for up to 3 candidate images per view, ranked by quality. All downloads appear in the gallery tagged by run number. Images that fail the quality gate receive visual indicators but are never deleted.
